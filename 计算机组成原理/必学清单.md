# 一、计算机五大核心部件
- **运算器**：做计算（加减乘除、逻辑判断）
- **控制器**：指挥各部件协作（CPU 核心）
- **存储器**：存数据 / 代码（内存、硬盘）
- **输入设备**：键盘、鼠标、触屏
- **输出设备**：屏幕、音箱

# 二、CPU（中央处理器）
### 核心作用
电子设备的大脑，所有程序运行、数据计算、硬件调度全都由它负责。

### 三大核心内部组件
- **运算器 ALU**
专门做计算：加减乘除、大小判断、逻辑判断。
- **控制器 CU**
总指挥，负责读取指令、安排执行顺序、协调所有硬件配合工作。
- **寄存器**
CPU 自带极速临时小仓库，存放正在处理的数据，速度远超内存。

### CPU 工作四步流程
频率越高，处理速度越快，代码执行、页面渲染就更流畅。
1. 取指令：从内存读取需要执行的命令
2. 译指令：把命令翻译成 CPU 能识别的语言
3. 执行运算：运算器完成数据处理
4. 结果写回：处理完的数据放回内存 / 缓存

### 高频关键参数
1. 核心数
物理核心，相当于独立大脑
- 单核：一次只能干一件大事
- 多核：多个大脑同时干活，多开软件、剪辑、后台任务更快
2. 线程
超线程：1 个物理核心模拟成 2 个逻辑线程，提升并发处理能力
例：4 核 8 线程、6 核 12 线程
3. 主频（GHz）
CPU 每秒运算次数，主频越高，单软件运行速度越快
4. 三级缓存 L1/L2/L3
层级越小速度越快、容量越小优先读取缓存数据，大幅减少等待内存的时间

### 经典比喻（超好记）
- CPU = 人脑（负责思考计算）
- 内存 = 办公桌（临时摆放要用的资料）
- 硬盘 = 仓库（长期存放所有文件软件）

# 三、存储器层级
- 寄存器（CPU 内）：最快，容量极小
- CPU 缓存（L1/L2/L3）：次快，减少 CPU 等内存
- 内存（RAM）：运行程序用，断电丢失
- 硬盘（ROM/SSD）：永久存储，容量大

# 四、内存（RAM）
1. 作用
代码、页面、图片、变量运行时必须加载到内存，才能被 CPU 执行。
2. 特点
- 速度快
- 断电 / 关闭程序，数据全部丢失
- 内存不足 → 浏览器卡顿、页面崩溃、OOM（内存溢出）
3. 和硬盘的区别
- 内存：临时运行
- 硬盘：永久保存

# 五、硬盘：永久存数据的 “仓库”
- 机械硬盘 HDD：便宜、容量大、速度慢
- 固态硬盘 SSD：速度快、贵、前端项目部署 / 加载全靠它
- 前端场景：
  项目代码、图片、静态资源都存在硬盘，运行时加载到内存。

# 六、总线：计算机的 “高速公路”
3 种总线（只记作用）
- 数据总线：传数据（JS 变量、网络数据、图片）
- 地址总线：定位数据存在哪（内存 / 硬盘地址）
- 控制总线：发指令（开始 / 停止 / 读写）
👉 带宽越大，数据传输越快，页面加载、接口请求速度越快。

# 六、计算机单位
1. 最小单位
- bit（比特）：1 个 0 或 1，最小单位
- Byte（字节）：8bit = 1Byte（1B）
2. 常用换算
1 KB = 1024 B
1 MB = 1024 KB
1 GB = 1024 MB

# 八、二进制与编码
1. 计算机只认二进制：0 和 1
2. 编码：把文字 / 符号转成二进制
- ASCII：英文 / 数字
- UTF-8：前端通用，支持中文（1 个中文占 3 字节）
3. 前端场景：
页面加 <meta charset="UTF-8"> 防止中文乱码。

#九、I/O 输入输出（代码和硬件交互）
1. I/O 操作很慢（远慢于 CPU / 内存）
网络请求、文件读取、硬盘写入都属于 I/O
2. 前端异步原理：
JS 单线程遇到 I/O 不阻塞，交给操作系统处理，完成后回调 → 这就是事件循环底层逻辑。
